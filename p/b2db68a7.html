分布式系统

第1章 概论

分布式计算源于计算机网络，但不同于计算机网路，是建立在计算机网络之上的技术。

1.1 分布式计算的定义与挑战

1.1.1 分布式计算系统的定义

**集中式：**一个系统的部件局限在一个地方

**分散式：**一个系统的部件在不同地方

  • **网络系统：**分散式系统的部件之间不存在或仅存在有限的合作
  • **分布式系统：**分散式系统的部件之间存在紧密的合作

分布式系统可以用硬件、控制和数据三个维度加以检验：

分布式计算系统 = 分布式硬件 + 分布式控制 + 分布式数据

  1. 系统是由多个计算机或计算机族（集群）组成，这些计算机或集群可以是异构的。
  2. 这些计算机在物理上是独立的，在地理上是分散的， 计算机运行其自身的操作系统，称为局部操作系统。局部操作系统也可以是异构的。一个作业或程序可以在一个或多个计算机节点运行，每个节点具有自治能力。
  3. 各计算机的地位是平等的。除了受全局分布式操作系统或中间件协同工作外，不存在或很少有主从控制或某种集中控制环节。
  4. 这些计算机组成一个整体，对用户是透明的，尽力呈现出单一系统试图。（即底层运行可能来自多个系统及设备的协同，但是操作的用户只知道自己在操作当前系统。）

应用实例： 大学或公司部门的工作站网络系统、工作信息流系统、基于 Web 网的一些服务模型。

1.1.2 分布式计算系统的要求

要求：

  1. 开放性
  2. 可扩展性
  3. 异构性
  4. 透明性
  5. 安全性

1. 开放性

能够用各种方法进行扩展或重新实现，即新的共享资源或服务能否被加入并为各客户程序所利用。

分布式计算系统是由来自不同厂商的异构的硬件和软件组成，它的开放性应该是基于统一的通信协议和遵循公开的访问共享资源的标准化接口。

2. 可扩展性

一个系统在资源数量和用户数量增加时，仍能有效工作。

3. 异构性

Internet 允许用户访问运行在异构计算机和网络上的服务，用户程序可以运行在这类异构的结构上。

异构表现在：

  1. 异构网络。Ethernet、Token_ring、ATM 等不同的网络，它们的差异由 Internet 的 TCP（UDP）/IP 和其他应用层协议予以隐藏。
  2. 异构计算机硬件。诸如整数类型在不同结构的计算机上有不同的表示（字节序不同等）。
  3. 异构操作系统。不同的操作系统可能为应用程序提供不同的接口。例如在 UNIX 上调用消息交换是不同于 Windows NT。
  4. 异构程序设计语言。不容程序设计语言的字符和数据结构的表示也不同。
  5. 不同开发商的实现。可能不使用统一的标准。

分布计算系统隐蔽异构性的主要办法是：

  1. 中间件技术（Middleware）。中间件是一种软件层，它提供程序设计的抽象，掩盖底下网络、硬件、操作系统和程序设计语言的差别。
  2. 虚拟机技术。提供一种方法，使代码在任何硬件上都能运行。

4. 透明性

向用户和应用程序隐藏了分布式计算系统部件的差异，系统被认为是一个整体，而不是独立部件的集合。

5. 安全性

  1. 身份认证技术
  2. 消息加密技术
  3. 访问控制技术

1.1.3 分布计算系统的应用

  1. 科学计算和高性能应用
  2. 现代跨企业的电子商务应用
  3. 容错应用
  4. 固有的分布式应用

1.2 分布计算系统的互联

1.2.1 共享存储器多处理机系统

共享存储器多处理机系统中所有 CPU 直接访问共享的存储器。

它可分为：

  • 基于总线的多处理机
  • 基于交换的多处理机

1. 基于总线的多处理机系统

基于总线的多处理机是用一条高速总线将多台 CPU 和一个共享存储模块连接在一起。

如果高速总线上挂载处理机数量较多时，总线效率将急剧下降。解决方法：可以给每个处理器添加缓存，如果缓存足够大，那么总线的负担就会小，从而可以挂载更多的处理器。（不过引入缓存，又引入了非一致性问题。）

2. 基于交换的多处理机系统

尽管基于总线的多处理机系统采用了高速缓存，但是挂载 CPU 的数量还是有限。譬如说， CPU 在 256 个以上，互联机制就要采用交叉开关或 Omega 交换网等高速互联机制。需要将存储器分成多个模块，形成二维布局。如果 CPU 和存储器模块都是 n 个，最好情况是形成 n 个并行通路， n 个 CPU 可同时访问给自的存储模块。

（b）的优点是开关数目少，但信号传送级数增多，延时增大。

**一致性存储访问结构 UMA（Uniform Memory Access）：**基于总线或基于交换的多处理机：

  • 对称多处理机系统：所有处理机都能执行操作系统和 I/O 操作
  • 非对称多处理机系统：只有特定的处理机执行操作系统和 I/O 操作

非一致性存储访问结构 NUMA（Non-Uniform Memory Access Architecture）： 每个处理机都有私有存储器而这些私有存储器又声称可被其他处理机共享（构成全局存储器）。

1.2.2 同构多计算机系统

与多处理机系统不同，组建同构多计算机系统相对容易些，因为每个节点是一台计算机，包含 CPU 和存储器。唯一要解决的就是节点之间的通信问题。与共享存储器多处理器的 CPU 和存储器之间的通信相比，节点间的通信量要少几个数量级。

  1. 基于总线的多计算机系统。这种系统规模较小，采用高速以太网或 Myrinet 互连多个工作站，称为工作站集群 COW（Cluster Of Workstations）。它没有专门措施来保证高速的 I/O 带宽，也没有专门的故障防护措施，使得 COW 价格低廉，易于使用。
  2. 基于交换的多计算机系统。这种系统规模很大，通常有几千个节点，称为大规模并行处理机（MPP）系统。

无论是规模较小的工作站集群，规模很大的并行高性能计算机，都可以接入 Internet，成为分布式计算系统的一个节点。

1.2.3 异构多计算机系统

目前，多数分布式计算系统是建立在异构多计算机系统之上。组成异构多计算机的节点可能有着很大的差异，表现在处理机类型、存储器容量大小、I/O 带宽以及操作系统等方面。有的节点本身就可能是多处理机系统、集群或并行高性能计算机。异构多计算机系统节点间是通过互联网络如 Internet 连接起来的。计算机网络实际上是异构多计算机系统。

1.3 分布式操作系统

1.3.1 分布式操作系统的组织

传统上，操作系统是用来管理单 CPU 计算机的。操作系统的主要目的是让多任务的应用程序方便地共享资源，这些资源包括 CPU、主存储器、磁盘、打印机、进程和文件系统。共享资源意味着不同的应用程序可用彼此隔离的方式访问同一个资源，就像这个资源被一个应用程序独占一样。这种隔离机制是采用信号灯和临界区。

操作系统必须对资源的使用方式进行完全控制，因此，大多数 CPU 支持两种操作模式：

  • 内核模式：允许执行 CPU 的所有指令
  • 用户模式：执行用户程序，对存储器和设备寄存器的特权指令是不能使用的（若有特殊需求，可以切换至内核模式再执行。）

操作系统有两种结构：

  • 单晶结构：整个操作系统的代码是一个庞大的单一的程序，全部驻留在内核地址空间

    效率高但修改与调试困难。

  • 微内核结构：将操作系统分成两个部分。

    • 管理模块：如虚拟存储器管理模块、进程管理模块和文件（设备）管理模块，它们由专门的服务器管理。
    • 小型微内核：驻留在主存中，是截获硬件中断和系统调用、从用户模式到内核模式的上下文切换、设置设备寄存器、重定向系统调用和中断到相应的管理模块。

    微内核结构对管理模块的修改和升级方便，但效率可能低一些。

1. 组织方法

  1. 纵向组织：

    用户进程与内核进程没有什么差别。用户进程提供缺陷或系统调用进入内核，内核执行完请求后返回用户进程。内核的资源由数据结构表示，为进程所共享。

    纵向组织更适合 UMA 多处理机结构。

  2. 横向组织：

    每个内核资源用一个内核进程表示，这些内核进程的同步和应用进程与内核进程间的通信是通过消息传送机制实现的。

    横向组织更适合分布式多计算机结构。

2. 硬件结构

根据硬件组织结构的不同，分布式操作系统大致又分为：

  1. 多处理机操作系统：
    • 支持并行高性能计算。
    • 共享存储器中的数据被多个处理机访问，要求并发控制，并发控制机制可以是信号灯（semaphore）、监控器（monitor）或加锁（lock）。
    • 多处理机对用户是透明的，系统表现为单一视图，用户不知道有多个处理机存在。
    • 通信是用共享存储器实现。
    • 操作系统和 I/O 处理所有处理机执行或由指定的处理机执行，分别称为对称多处理机系统或非对称多处理机系统。
  2. 多计算机操作系统：
    • 支持并行高性能计算。。
    • 系统由多台计算机通过高速互连组成，对用户表现为单一视图，即用户不知道有多个计算机存在。
    • 每台计算机都有自己的操作系统内核，执行分布式操作系统的服务。
    • 没有物理的共享存储器，为了执行在共享存储器多处理机上开发的软件，多计算机操作系统往往包含由软件实现的分布式共享存储器。
    • 各计算机进程之间的通信采用消息传送机制，不同的系统，消息的语义可能不同。
